# 需要询问的输入

> 当用户给出的信息不足以支撑决策时，按此清单逐一确认。

## 确认清单

| 问题 | 默认行为 | 何时需要询问 |
|:-----|:---------|:------------|
| **单个提交还是多个提交？** | 当存在不相关的变更时，默认为多个小提交。如果变更统一聚焦，默认为单个提交。 | 变更分散且不确定用户意图时。如果用户说"一起提交"则跳过。 |
| **提交风格** | 必须使用 Conventional Commits，并包含 Emoji | 无需询问（技能已定义）。 |
| **任何自定义规则** | 最大标题长度、必需的作用域等 | 仅当 commitlint 规则缺失且用户行为模式不明时询问。 |

## 沟通原则

- 优先通过上下文推断：检查 git log 历史推断用户习惯
- 只在确实无法推断时才提问
- 每次至多问 1-2 个问题，不要让用户面对太多选择
