# versionMode 决策

## 语义澄清

| 需求表述                                               | 推荐 `versionMode`                    | 说明                         |
| ------------------------------------------------------ | ------------------------------------- | ---------------------------- |
| 各子包**独立版本线**（历史版本互不强制对齐）           | `independent`                         | 与「独立版本」语义一致       |
| 只 bump **有变更的包**，但本次发布**共用一次版本语义** | 常是 `selective` 或工具组合语义       | **不等于**「每包独立版本线」 |
| 全仓统一版本号                                         | `unified`（若 relizy 支持且团队接受） | 需与仓库现状一致             |

## 常见误判

- **错误**：把 `selective` 当成「独立版本」。
- **正确**：`selective` 关注「哪些包参与本次 bump」，不解决「每包一条独立版本线」。

## 阻断规则

若用户明确要独立版本线，却拟配置 `selective` 作为替代，**必须停止**并回到上表澄清。
