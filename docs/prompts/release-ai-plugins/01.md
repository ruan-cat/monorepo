## 037 <!-- 已完成：init-prettier-git-hooks / init-vscode 增补 LF EOL 合并链路 --> 拓展 `init-prettier-git-hooks` 技能和 `init-vscode` 技能

我在多个项目内都遭遇到了很难受的 git 幽灵修改问题，我需要让 `init-prettier-git-hooks` 技能做好文件 eol 行尾字符的处理。

目前完整完善的处理链路如下：

1. `.gitattributes`：`* text=auto eol=lf`
2. `.editorconfig`：`end_of_line = lf`
3. `prettier.config.mjs`：`endOfLine: "lf"`（**禁止** `"auto"`，`"auto"` 在 Windows 上会保留/引入 CRLF）
4. `.vscode/settings.json`：`"files.eol": "\n"`
5. 运行 `git add --renormalize .` 实现 git 文件归一化。

我需要让项目拥有这些能力，初始化配置时，同时确保处理了 git 的修改问题。重点统一 eol 配置为 `lf` 。

### `init-prettier-git-hooks` 技能的职责

#### 明确合并策略为合并策略

不仅仅是初始化配置文件了，而且还要实现配置文件的动态修改，覆盖，拓展，补全。以`init-prettier-git-hooks` 技能提供的模板为例子，去补充，改写目标项目的配置项。因为你需要处理的目标项目，很多本身都已经提供了 `.gitattributes` `.editorconfig` 和 `prettier.config.mjs` 这些配置文件，但是你要做到更新和拓展。所以你不应该再无条件覆盖了，而是动态识别与合并。

比如目标项目已经存在了 `prettier.config.mjs` 配置，但是 `endOfLine: "auto",` ，不符合我们现在的 `endOfLine: "lf",` 所以你应该用 lf 去精准覆盖掉 auto 配置。

#### 调研并准备一份合适的 `.gitattributes` 和 `.editorconfig` 配置

我需要你先调研本机电脑上其他项目是怎么配置编写 `.gitattributes` 和 `.editorconfig` 配置的，然后再确保能完成 eol 为 lf 的情况下，在 `ai-plugins\common-tools\skills\init-prettier-git-hooks\templates` 目录内补全模板。

### `init-vscode` 技能的职责

专门负责 `.vscode/settings.json`：`"files.eol": "\n"` 的配置。`init-vscode` 技能专门实现 vscode 配置的处理，所以由 `init-vscode` 技能拓展这个配置。
